# User Stories

This document outlines the user stories for TaskTeam, organized by feature area.

## 1 Authentication 

US-1: As a new user, I want to sign up using an email and password, so that I can securely create an account. 

US-2: As a returning user, I want to log in to my account, so that I can access my teams, projects, and tasks. 

US-3: As a logged-in user, I want to log out, so that my account remains secure when I am finished using the app. 

## 2 Team Management 

US-4: As a user, I want to create a new team, so that I can collaborate with others in a shared workspace. 

US-5: As a user, I want to join an existing team, so that I can work with classmates or coworkers. 

US-6: As a user, I want to view all teams I belong to on a dashboard, so that I can easily switch between teams. 

US-7: As a user, I want to leave a team, so that I am no longer part of inactive or irrelevant projects. 

## 3 Team Chat → quick, real-time conversation 

US-8: As a user, I want to send real-time messages in a team chat, so that I can communicate quickly with my teammates.  

US-9: As a user, I want to view messages sent by other team members in real time, so that conversations feel immediate. 

US-10: As a user, I want to see who sent each message and when, so that conversations are clear. 

## 4 Message Management 

US-11: As a user, I want to scroll through recent chat history, so that I can catch up on conversations I missed. 

US-12: As a user, I want to delete or edit my own chat messages, so that I can correct mistakes. 

## 5 Project Management 

US-13: As a user, I want to create a project within a team, so that work can be organized around a specific goal. 

US-14: As a user, I want to view all active projects for a team, so that I can focus on current work. 

US-15: As a user, I want to view archived projects, so that I can reference completed or paused work. 

US-16: As a user, I want to archive a project, so that it no longer appears in the active projects list. 

## 6 Task Management 

US-17: As a user, I want to create tasks within a project, so that work items are clearly defined. 

US-18: As a user, I want to move tasks between To Do, In Progress, and Done, so that progress is easy to track. 

US-19: As a user, I want to assign tasks to team members, so that responsibilities are clear. 

US-20: As a user, I want to add details to a task (description, priority, checklist), so that expectations are well defined. 

US-21: As a user, I want to set due dates and priorities for tasks, so that deadlines are clear. 

## 7 Discussions → structured, persistent project-level posts 

US-22: As a user, I want to create a discussion post within a project, so that I can share updates or start conversations. 

US-23: As a user, I want to post daily memos, so that my team stays informed about progress or thoughts. 

US-24: As a user, I want to ask questions in the discussion forum, so that I can get help or clarification. 

US-25: As a user, I want to categorize discussion posts (Daily Memo, Question, Discussion, Blocker), so that conversations are easy to understand. 

US-26: As a user, I want to reply to discussion posts, so that conversations can happen in one place. 

US-27: As a user, I want to edit or delete my discussion posts, so that I can correct mistakes or remove outdated information. 

US-28: As a user, I want to pin important discussion posts, so that key information is easy to find. 

US-29: As a user, I want to search discussion posts, so that I can quickly find past conversations. 

## 8 Activity Feed 

US-30: As a user, I want to view a read-only activity feed, so that I can track changes made within a project. 

US-31: As a user, I want to see who performed actions such as creating tasks, editing discussions, or completing work, so that I have visibility into team activity. 

## 9 Notifications  

US-32: As a user, I want to receive notifications when a task assigned to me is created or updated, so that I stay aware of my responsibilities. 

US-33: As a user, I want to receive notifications when someone comments on or replies to a discussion I created, so that I can respond promptly. 

US-34: As a user, I want to receive notifications when a project discussion is marked as a blocker, so that I am aware of issues affecting progress. 

US-35: As a user, I want to view notifications in an in-app notification panel, so that I can quickly see recent updates without leaving the app. 

## 10 Notification Management 

US-36: As a user, I want to mark notifications as read, so that I can keep my notification list organized. 

US-37: As a user, I want notifications to link directly to the relevant task, discussion, or project, so that I can take action quickly. 

## 11 Nice-to-Have  

US-38: As a user, I want to switch between light and dark mode, so that the interface is comfortable in different environments. 

US-39: As a user, I want my UI preferences (theme, layout) to be saved, so that the app feels personalized. 

US-40: As a user, I want to see avatars for team members, so that collaboration feels more personal. 

US-41: As a user, I want to see when team members are online, so that I know who may be available to respond.
